# PRD Skill — 產品需求文件

這個 skill 會引導 AI agent 產出一份完整的 PRD（Product Requirements Document），幫助團隊在動手開發前對齊目標、功能範疇與使用者故事。

## When to use this skill

- 剛拿到一個專題題目，還不確定要做什麼功能
- 想把口頭討論的想法變成正式文件
- 開始寫程式之前，先確認大家對「要做什麼」有共識

## How to use it

請在你的 prompt 裡使用以下指示，讓 AI 產出 PRD：

```
請幫我產出這個專案的 PRD 文件，儲存為 docs/PRD.md。

專案名稱：[你的專案名稱]
專案描述：[一到兩句話說明這個系統要解決什麼問題]

PRD 文件需包含以下章節：

1. **專案概述**
   - 背景與動機
   - 目標用戶
   - 核心價值主張

2. **功能需求**
   - 列出所有主要功能（至少 5 項）
   - 每項功能用「作為 [角色]，我希望 [行為]，以便 [目的]」格式寫使用者故事

3. **非功能需求**
   - 技術限制（使用 Flask + Jinja2 + SQLite）
   - 效能與安全考量

4. **MVP 範圍**
   - 列出 Must Have / Should Have / Nice to Have 三個層級的功能

5. **專案成員與分工**
   - 留空白表格讓團隊填寫

請用繁體中文撰寫，格式清楚，適合給開發團隊參考。
```

## 產出範例

執行後應在 `docs/PRD.md` 看到結構完整的需求文件，包含使用者故事與 MVP 功能清單。
